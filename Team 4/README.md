# Final Submission README

## What created `submission.csv`?

The current final file `submission.csv` is generated by:

- `train_model.py`

This is the only local Python code file in this repository that writes `submission.csv`.

Direct code evidence:

- `train_model.py` reads `train.csv`, `test.csv`, and `sample_submission.csv`
- `train_model.py` saves `submission.csv`

## Files used for the final `submission.csv`

### Local code files used directly

- `train_model.py`

### Data files used directly

- `train.csv`

### Files produced during execution

- `submission.csv`

## Final modeling pipeline in `train_model.py`

`train_model.py` builds the final `submission.csv` with the following pipeline:

1. Load `train.csv`, `test.csv`, and `sample_submission.csv`
2. Replace numeric `-1` values with `NaN`
3. Add global engineered features:
   - `req_rate_ratio_6h_24h`
   - `req_rate_ratio_24h_4w`
   - `addr_diff`
   - `device_email_ratio`
   - `income_per_age`
   - `prev_addr_missing`
   - `bank_months_missing`
   - `req_rate_acceleration`
   - `total_reqs`
4. Detect categorical columns and cast them for model training
5. Run 5-fold stratified cross-validation
6. Inside each fold, build fold-safe aggregation features to avoid leakage:
   - target encoding for selected categorical columns
   - grouped income statistics
   - grouped credit score statistics
7. Train two models on each fold:
   - LightGBM
   - CatBoost
8. Average the fold test predictions from both models
9. Write the ensemble result to `submission.csv`

`submission_cat.csv` is also saved from the same run as the CatBoost-only prediction.

## Environment

Recommended:

- Python 3.10 or newer
- CPU execution is enough for the current code

Required Python packages:

- `pandas`
- `numpy`
- `scikit-learn`
- `lightgbm`
- `catboost`

Example setup:

```bash
python -m venv .venv
source .venv/bin/activate
pip install pandas numpy scikit-learn lightgbm catboost
```

## How to run

From the repository root:

```bash
python train_model.py
```

## Expected outputs

After the run finishes, these files should exist or be updated:

- `submission.csv`
- `submission_cat.csv`
- `catboost_info/`

## Important notes

- Running `train_model.py` will overwrite the existing `submission.csv` and `submission_cat.csv`.
- The code uses fixed seeds (`random_state=42` / `random_seed=42`) for reproducibility.
- The final `submission.csv` is an ensemble of LightGBM and CatBoost predictions.
